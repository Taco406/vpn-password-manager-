# NorthKey — iPhone app

The NorthKey vault in your pocket: scan the QR from the desktop app to connect, unlock with the
**same master password** you use on your computer (Face ID optional after that), and view, add,
and edit your passwords — synced end-to-end-encrypted through your own sync server, exactly like
the desktop apps.

> **Not built in CI.** This directory is complete SwiftUI + vendored-C source, an
> [XcodeGen](project.yml) spec, and setup docs. It needs a Mac with Xcode to build — there is no
> Swift toolchain in the NorthKey CI environment. Byte-compatibility with the desktop's Rust crypto
> is enforced by unit tests against committed golden vectors (`NorthKeyTests/Fixtures/`,
> generated by `crates/core/tests/ios_golden_vectors.rs`) — run them with **⌘U** (simulator is
> fine).

## What it does

- **Scan to connect** — in NorthKey on your computer: *Account & Sync → Add a device*. Scan the QR:
  it carries the server address, the exact TLS certificate to pin, and a one-time 5-minute
  enrollment code. No tokens to type (a manual server-URL + setup-token path stays under
  Advanced).
- **One login everywhere** — enter your account master password; the phone downloads the escrowed
  wrapped key (`SNTL` type-4), unwraps it locally with Argon2id, and decrypts the same `SVLT`
  vault blob your desktops sync. The password and keys never leave the phone.
- **Real vault client** — browse and search items, copy usernames/passwords (sensitive copies are
  local-only and expire after 60 s), live TOTP codes for items that have one, add/edit logins and
  secure notes, delete with tombstones. Pushes use the same optimistic-concurrency `PUT /v1/vault`
  as the desktop.
- **Face ID (optional)** — after the first unlock, turn on *Unlock with Face ID next time* (menu in
  the vault list). The vault key is stored in the Keychain behind `.biometryCurrentSet`.

## Crypto (mirrors the Rust core 1:1)

- Argon2id KEK (m=64 MiB, t=3, p=4) via the vendored reference C implementation
  (`Vendor/argon2`, built with `ARGON2_NO_THREADS`).
- XChaCha20-Poly1305 = HChaCha20 subkey (implemented in Swift, vector-tested) + CryptoKit's IETF
  ChaChaPoly — the exact construction of the Rust `chacha20poly1305` crate.
- HKDF-SHA256 with the `sentinel/v1/vault/*` info strings (protocol constants, checked by
  `just ios-docs-check`).
- zstd via the vendored C library (`Vendor/zstd`, `ZSTD_DISABLE_ASM`).
- TLS: exact-certificate pinning against the QR-delivered PEM (`CertPinDelegate`) — the desktop
  equivalent of its pinned reqwest client.

## Building it (on a Mac)

1. Generate and open the project:
   ```bash
   brew install xcodegen
   cd apps/ios-key && xcodegen generate && open NorthKey.xcodeproj
   ```
2. Set your **Team** (Signing & Capabilities) and, if `com.northkey.app` is taken, a unique bundle
   id. A free Apple ID can sideload to your own iPhone (7-day certificate); push notifications are
   the only feature that needs the paid program.
3. **⌘U** runs the crypto golden-vector tests (simulator OK).
4. Build to your iPhone, then scan the desktop's *Add a device* QR.

## Layout

```
apps/ios-key/
  project.yml                  XcodeGen spec (app + unit-test targets, vendored C)
  Config/Info.plist            usage strings, background modes
  Config/NorthKey.entitlements push + data-protection
  Vendor/argon2, Vendor/zstd   reference C, compiled into the app
  NorthKey/
    NorthKeyApp.swift          app entry
    ContentView.swift          the three phases: connect / unlock / vault
    Bridging.h                 exposes the vendored C to Swift
    Onboarding/ScanSetupView.swift    QR-first onboarding (+ Advanced manual path)
    Onboarding/ServerSetupView.swift  typed-address sign-in (master password + trust check)
    Api/ApiClient.swift        pinned-TLS client, enroll/vault/wrapped-key endpoints
    Crypto/VaultCrypto.swift   Argon2id + XChaCha20-Poly1305 + HKDF + zstd bridge
    Vault/VaultModels.swift    Codable mirrors of the Rust vault model
    Vault/VaultStore.swift     unlock / pull / edit / push session
    Vault/VaultViews.swift     unlock screen, list, detail, add/edit
    Pairing/, Approvals/, Totp/  earlier approver/pairing companion (kept for the
                                 upcoming approve-on-phone feature; not in the main flow)
  NorthKeyTests/
    VaultCryptoTests.swift     golden-vector byte-compat tests (⌘U)
    Fixtures/golden-vault.json generated by crates/core/tests/ios_golden_vectors.rs
  PROJECT.md                   exact target/entitlement/capability settings
```
